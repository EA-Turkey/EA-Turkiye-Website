## Architecture

Hugo static site, no theme, no JS framework, no CMS. All layout logic lives in root-owned `layouts/`.

**Bilingual structure.** Turkish at `/`, English at `/en/`. Each translated page is a pair sharing one `translationKey`; internal folder names match across languages, only public `slug`/`url` localizes. Turkish source: `content/tr/`. English source: `content/en/`. Site-wide UI strings: `i18n/{tr,en}.yaml`.

**Hugo's `contentDir` language bug is treated as active** (see AGENTS.md). When working in templates:
- Detect active language from the URL
- Resolve current-language menus and site data via `site.Sites`
- Do NOT rely on raw `.Site.Language`, raw `.Site.Menus.main`, or `.Section`

**Image pipeline.** Source photos go in `assets/images/...` and are referenced in front matter without that prefix (e.g. `featured_image: events/foo/cover.jpg`). Responsive AVIF + MozJPEG derivatives are committed under `static/avif/` and `static/jpg/`, generated by the two scripts above. `layouts/partials/picture.html` consumes them. Brand assets (logos, icons) that should pass through unchanged go in `static/brand/`.

**Shared data.** Repeated people data lives in `data/team.yaml` (consumed by About / Get involved). Sitewide links (contact email, Luma calendar URLs, social) live in `hugo.toml` `[params]`. Header/footer/hero/etc. markup lives in `layouts/partials/`.

**Search.** Search source remains dormant. The `searchIndex` output (`index.searchindex.json`, served as `/search-index.json`), overlay partial, and vanilla JS remain in the repo for possible future use, but `baseof.html` does not render the overlay or load `assets/js/search.js`; dormant UI styles live in `assets/css/search.css` and are not included by `site-styles.html`.

**Contact form.** Posts to `/api/contact`, implemented as a Cloudflare Pages Function in `functions/api/contact.js`. Submissions are stored in D1 via `CONTACT_DB`; D1 is the source of truth. After a successful insert, the function can send a Resend notification to the single fixed recipient `bilgi@eaturkiye.org`. The submitter never controls the notification recipient and no autoresponder is sent. Required runtime configuration is documented in `README.md`; do not commit `RESEND_API_KEY`, `.env`, or `.dev.vars`.

**Deployment.** Cloudflare Pages, output `public/`. `cleanDestinationDir = true` is set, so committed `public/` artifacts are not authoritative.

## Hard rules (do not break)

- Do not reintroduce Doks/Thulite, another theme, Hugo Modules, or a Node-based build step.
- Do not add a CMS, a JS framework, or build-time external fetches.
- Do not split `hugo.toml` back into multiple config files.
- Do not hardcode `/en/` or Turkish URLs in templates â€” use `pageRef`, language-aware lookups, or translation pairs.
- Do not duplicate header/footer markup across templates â€” edit `layouts/partials/`.
- Do not move processable photos into `static/`.
- When editing content, edit both the Turkish and English file in a translation pair unless the user explicitly says otherwise.
